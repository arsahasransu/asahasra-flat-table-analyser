# ML Data Preparation (ROOT to Torch)

This directory contains scripts and notebooks designed to prepare analysis datasets for Machine Learning pipelines. Specifically, it extracts data from ROOT trees (`snapshot` ntuples generated by the analyser), processes it into flattened Awkward arrays, and outputs train/test splits for ML models (e.g., in PyTorch).

## Files Overview

- **`prepare_data.py`**
  The main Python script containing the logic for extracting, filtering, and splitting the data. It leverages `uproot` and `awkward` array operations to manipulate nested ROOT branch data into a format suitable for ML model consumption.
  
- **`prepare_data.ipynb`**
  A Jupyter Notebook used for prototyping the data loading and processing workflow implemented in `prepare_data.py`.

## Core Logic & Functions

### `prepare_data.py`

1. **`flatten_per_tkel(filename, tkeltag, puppitag)`**
   - Opens the `snapshot` tree from the specified `.root` file using `uproot`.
   - Extracts Reconstructed Track Electrons (`TkEleL2`) and L1 PUPPI candidates into separate Awkward arrays.
   - Computes a Cartesian product of the two collections to evaluate $\Delta\eta$, $\Delta\phi$, and $\Delta R$.
   - Filters PUPPI candidates surrounding the electron within an annulus/cone mask ($0 < \Delta R < 0.5$).
   - Returns a flattened array where each record corresponds to a single Track Electron, containing its properties and a nested array of its matched PUPPI candidates (`mpuppi`).

2. **`split_puppi_by_pdgid(data)`**
   - Takes the output from `flatten_per_tkel` and categorizes the nested `mpuppi` collection by the absolute value of their `pdgId`.
   - Produces distinct collections for:
     - Electrons (`pdgId` 11, `mpuppiel`)
     - Photons (`pdgId` 22, `mpuppiph`)
     - Muons (`pdgId` 13, `mpuppimu`)
     - Charged Hadrons (`pdgId` 211, `mpuppichg`)
     - Neutral Hadrons (`pdgId` 130, `mpuppineu`)

3. **`prepare_ml_data(signal_file, bkg_file)`**
   - Processes the signal file (e.g., DY analysis file) mapping it to `label = 1`.
   - Processes the background file (e.g., MinBias analysis file) mapping it to `label = 0`.
   - Merges, shuffles, and yields an 80:20 training and testing split.

## Usage

To generate the datasets directly, run the script from this directory:

```bash
python prepare_data.py
```

By default, the script looks for `DY_PU200_EB_snapshot.root` (signal) and `MinBias_EB_snapshot.root` (background) in the top-level repository directory (e.g., `../../`). Adjust the paths in the `if __name__ == "__main__":` block to process different snapshot combinations or pseudo-rapidity regions (e.g., EE instead of EB).
